## 2. 系统能力总览

### 2.1 指令处理分层（设计原则）

所有语音输入按三类分流，避免「控制口令」被误当成「创作描述」：

| 层级 | 含义 | 处理位置 | 示例 |
|------|------|----------|------|
| L1 导航口令 | 切换页面/模式 | 前端本地 | 「AI 画图」「图表」「画板」 |
| L2 操作口令 | 控制流程，不触发生成 | 前端本地 + 后端兜底 | 「下载图片」「返回主菜单」「关闭语音」 |
| L3 创作描述 | 触发生图/图表流水线 | 上传后端 | 「画一只在月亮上的猫」 |

### 2.2 语音技术架构（最终实现）

| 组件 | 作用 |
|------|------|
| Web Speech API | 常驻识别，实时预览（`live` 区域），本地口令匹配 |
| VAD + MediaRecorder | 检测有效语音段，静音 3s 收口后录音 |
| 百度短语音 REST ASR | 正式提交音频转文字（浏览器内 webm→16kHz WAV） |
| TTS（浏览器语音合成） | 状态播报；播报期间暂停识别，防自听 |

| 计划能力 | 触发示例 | 状态 | 实现说明 |
|----------|----------|------|----------|
| 语音描述生图 | 自然语言描述画面 | ✅ | VAD 收口 → 百度 ASR → LLM 润色 → 千帆生图 |
| 多轮润色 | 「在刚才基础上加…」 | ✅ | 自动携带 `prior_prompt`（`historyPrompt`） |
| 文本框提交 | 手动输入后点发送 | ✅ | `POST /api/voice/describe?mode=image` |
| 开启/关闭语音 | 「开启语音」「关闭语音」 | ✅ | 悬浮球可点；口令含同音纠正与防抖 |

| 完成后自动开麦 | （系统自动） | ✅ | TTS 结束后 `resumeVoiceAfterTask`，提示可继续说 |
| 返回主菜单 | 「返回主菜单」「回首页」 | ✅ | 不提交生成，直接回 `/` |
| 下载图片 | 「下载图片」「保存图片」 | ✅ | 本地三层拦截 + 后端 `VOICE_DOWNLOAD_IMAGE` 兜底 |

| 刷新画板列表 | 「刷新」 | ✅ | 调用 `fetchList()` |

| TTS 状态播报 | 生成完成 | ✅ | 「已经理解你的描述，图片已生成。」 |

## 4. 未完成项及原因说明

### 4.1 「停止 / 取消」中断正在进行的生成

| 项 | 说明 |
|----|------|
| **计划** | 用户说「停止」可取消当前上传或生图请求 |
| **状态** | ❌ 未实现 |
| **原因** | `fetch` 请求未接入 `AbortController`；千帆生图在后端同步等待，取消需前后端协同改造。Demo 阶段用「关闭语音」+ 生成锁 `generatingLock` 防止重复提交，优先级低于主链路交付。 |
| **后续** | 前端 `AbortSignal` + 后端任务 ID + 可取消的 httpx 调用 |

### 4.2 画板库页（`drawings.html`）内语音操作

| 项 | 说明 |
|----|------|
| **计划** | 在画板库内语音「删除第 3 条」「打开第 1 条」 |
| **状态** | ❌ 未实现（主页可说「画板」进入，库内无麦） |
| **原因** | 画板库为辅助管理页，语音加载/删除已在 `image.html` / `chart.html` 内嵌列表实现；独立页再复制一套 VAD 状态机成本高、演示主路径不经过该页。 |
| **后续** | 抽取公共 `voice.js` 后可在 `drawings.html` 复用 |

### 4.3 语音「保存到画板」（区别于自动入库）

| 项 | 说明 |
|----|------|
| **计划** | 口令保存当前结果到画板库 |
| **状态** | 🔶 部分实现 |
| **原因** | 生成后**已自动入库**，无需再口述保存；额外提供按钮「保存当前图片/图表」手动覆盖标题。未单独做「保存到画板」语音口令，避免与「下载图片」「自动入库」语义冲突。 |

### 4.4 「保存到桌面」及导出审计字段

| 项 | 说明 |
|----|------|
| **计划** | 语音「保存到桌面」，数据库记录 `exported` / `export_target` |
| **状态** | 🗑️ 已裁剪，改为「下载图片」 |
| **原因** | 用户实际需要的是浏览器下载 PNG，而非写入操作系统桌面路径；原方案易造成「说了下载但没文件」的误解。已改为 Blob 下载 + 去除导出 API，降低复杂度。 |

### 4.5 旧版「语音画板」几何绘图指令

| 项 | 说明 |
|----|------|
| **计划** | 「画一个圆」「撤销」「清空」等 Canvas 矢量指令（`/api/asr-nlu`） |
| **状态** | 🗑️ 已裁剪 |
| **原因** | 训练营题目聚焦 **AI 生图 + 图表**，非手写矢量画板；旧 `frontend/index.html` + NLU 链路已移除，避免双主线并行维护。 |

### 4.6 图表类型与多轮编辑深度优化

| 项 | 说明 |
|----|------|
| **计划** | 更多图表类型；多轮精确改某一数据系列 |

### 4.7 跨浏览器 / 离线 / 弱网完备性

| 项 | 说明 |
|----|------|
| **计划** | 全浏览器一致体验 |

### 4.8 仅润色提示词、暂不生成图片

| 项 | 说明 |
|----|------|

## 5. 关键体验设计（最终实现）

以下为开发过程中沉淀、已写入代码的交互规则：

| 规则 | 参数/行为 | 目的 |
|------|-----------|------|
| 最短有效语音 | ≥ 0.8s | 过滤咳嗽、噪声误触 |
| 静音收口 | 连续静音 3s | 避免半句话提交 |
| 生成冷却 | 完成后 3s | 防止连说触发两次 |
| 生成锁 | `generatingLock` | 上一张未完成不接受新单 |
| 口令防抖 | 800ms | 避免「关闭语音」连触发 |
| 口令窗口 | 最近约 96 字 | 长段识别不淹没短口令 |
| 下载防误触 | 前端 + 后端双拦截 | 「下载图片」不进入 LLM |
| 生成边界 | 自动关麦 → 自动开麦 | 用户明确「何时可继续说」 |

## 7. 附录：指令速查（最终实现）

### 主页
- 「AI 画图」「图表」「画板」「展开菜单」
- 「开启语音」「关闭语音」

### 生图 / 图表页
- **创作**：直接描述需求（停顿 3 秒自动提交）
- **控制**：「返回主菜单」「关闭语音」「开启语音」
- **下载**：「下载图片」/「下载图表」（或点按钮）
- **历史**：「刷新」「加载第 N 条」「删除第 N 条」「重试」

### 不建议说法
- 避免单独说「下载」二字（已收窄口令，但仍建议说全「下载图片」）
- 生成过程中（橙色「生成中」）请勿继续描述，等自动开麦后再说
